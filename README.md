# ターン制1v1対戦ゲーム

キャラクター1体がカードデッキのような複雑さを持つ、ターン制1v1対戦ゲーム。

## ゲームについて

このゲームは、ブラウザ上で遊べるターン制対戦ゲームです。ダウンロードやインストールは不要で、WebRTCによるP2P通信で友達とリアルタイム対戦ができます。

### 特徴

- **全キャラ使用可能**: 収集・育成・課金要素は一切なし。最初から全キャラクターを使えます
- **複雑なスキルシステム**: キャラごとに全く異質なスキル設計。1キャラを極める楽しさがあります
- **ポケモン形式**: 場に出ているのは常に1体のみ。控えの2体と戦略的に交代しながら戦います
- **勝利条件**: 相手チーム3体全滅で勝利
- **属性相性**: 火・水・草・光・闇の5属性による相性システム

## 遊び方

### 対戦モード

1. ホーム画面で「ルームを作る」をクリックし、ルームコードを取得
2. ルームコードを友達に共有
3. 友達が「ルームに参加する」でルームコードを入力
4. 両者が編成を確定した後、先頭キャラを選択して対戦開始

### ソロモード（CPU戦）

1. ホーム画面で「ソロモード」をクリック
2. 編成を選択
3. CPUと対戦

## ゲームルール

### ターン構造

1. **ターン開始**: コスト3を付与
2. **行動フェーズ**: 交代またはスキル使用（コストの範囲内で複数使用可能）
3. **効果処理**: スキル効果を順次処理
4. **敵撃破判定**: 敵を倒した場合、ターン終了
5. **ターン終了処理**: DoT・リジェネ適用、バフ・デバフ減少
6. **ターン交代**: 相手のターンへ

### 編成・交代ルール

- 各プレイヤーは3体のキャラクターでチームを組む
- 試合開始時に先頭に出すキャラ1体だけを選択
- 交代は原則スキル使用前のみ可能
- 死亡時は控えの2体からその場で選んで出す（30秒以内）

### スキルシステム

- 各キャラが持つスキル数は自由
- 派生スキル（特定スキル使用後に解放されるスキル）
- スキルシナジー（同一ターン内の組み合わせ効果）
- パッシブ能力（場に出ているキャラだけでなく、控えからも発動）

## 属性相性

| 攻撃側 | 有利な相手 |
|---|---|
| 🔥 火 | 🌿 草 |

有利属性へのダメージ: ×1.2

## キャラクター

現在実装されているキャラクター:

- **ルート.J**: 時間と死の宣告を操る機械
- **炎の剣士**: シンプルだが火力の高い剣士
- **氷の魔導士**: マナを消費して攻撃・回復を行う魔導士
- **時詠みの魔女**: 時計針を操作して時間を操る魔女
- **E-P-P**: 小隊を展開し、スキル効果を再使用する指揮官

詳細なキャラクター解説は [キャラクター解説.md](./キャラクター解説.md) をご覧ください。

### アーキテクチャ

#### バトルエンジン
- **サーバーレス設計**: ゲームロジックはホスト側のブラウザで処理
- **イミュータブルな状態管理**: すべての状態更新は新しい状態オブジェクトを返す
- **スキルハンドラパターン**: 各キャラのスキルは独立したハンドラ関数として実装
- **パッシブトリガーシステム**: 様々なタイミングでパッシブを発火可能

#### P2P通信
- **WebRTC**: ブラウザ間の直接通信
- **PeerJS**: シグナリングサーバーを使用して接続確立
- **ホスト・ゲストモデル**: ホストがゲーム状態を管理し、ゲストはアクションを送信

#### 状態管理
- **Zustand**: グローバルなアプリケーション状態を管理
- **ローカルストレージ**: プレイヤー名・編成設定などの永続化

### テスト方針

- **単体テスト**: バトルエンジンのコアロジックをテスト
- **コンポーネントテスト**: UIコンポーネントのレンダリングとインタラクションをテスト
- **統合テスト**: キャラクターのスキルハンドラが正しく動作することを確認

テストファイルは `src/lib/*.test.ts` および `src/test/` に配置されています。

### デプロイ

このプロジェクトは静的サイトとしてデプロイできます。

#### Vercelへのデプロイ

1. GitHubリポジトリを作成
2. Vercelでプロジェクトをインポート
3. ビルド設定: `npm run build`
4. 出力ディレクトリ: `dist`

#### その他のホスティング

- **Netlify**: ビルドコマンド `npm run build`、公開ディレクトリ `dist`
- **GitHub Pages**: `vite build` 後に `dist` ディレクトリを `gh-pages` ブランチにプッシュ

### トラブルシューティング

#### PeerJS接続が確立できない

- ファイアウォール設定を確認
- シグナリングサーバーの状態を確認（PeerJSの無料サーバーはダウンすることがあります）
- ブラウザのコンソールでエラーメッセージを確認

#### ビルドエラー

- Node.jsのバージョンを確認（推奨: 18.x 以上）
- `node_modules` を削除して再インストール: `rm -rf node_modules && npm install`

### 新キャラ追加の手順

1. `src/data/characters.json` にキャラデータを追加
2. `src/lib/skill-handlers/` にキャラ固有のハンドラファイルを作成
3. `src/lib/skill-handlers/index.ts` にハンドラをエクスポート

詳細は `docs/NEW_CHARACTER_GUIDE.md` を参照してください。
